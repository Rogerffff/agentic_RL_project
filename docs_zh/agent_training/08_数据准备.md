# 数据准备

本文档详细介绍 verl Agent 训练所需的数据格式、字段定义、预处理脚本以及自定义数据集的构建方法。

---

## 1. 数据格式概述

verl 使用 Parquet 格式存储训练数据。Parquet 是 Apache 开源的列式存储格式，具备高效压缩和快速读取的优势，被 Hugging Face `datasets` 库原生支持。

Agent 训练数据在标准 RLHF 数据基础上增加了 `agent_name`、`tools_kwargs`、`interaction_kwargs` 等字段，用于指定 Agent 类型、配置工具参数和交互参数。标准 RLHF 数据只需要 `prompt` 和 `data_source` 两个核心字段，而 Agent 训练还需要在 `extra_info` 中提供工具和交互的运行时参数。

数据流概览：

```
原始数据集 (HuggingFace / 本地)
    │
    ▼
预处理脚本 (examples/data_preprocess/)
    │  - 提取问题和答案
    │  - 构造 prompt (chat format)
    │  - 设置 agent_name, tools_kwargs, interaction_kwargs
    │  - 配置 reward_model
    ▼
Parquet 文件 (train.parquet / test.parquet)
    │
    ▼
verl RLHFDataset 加载
    │
    ▼
DataProto → Agent Loop → 训练
```

## 2. 数据字段说明

### 必需字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `prompt` | `list[dict]` | 初始对话消息列表，格式: `[{"role": "system", "content": "..."}, {"role": "user", "content": "..."}]` |
| `data_source` | `str` | 数据来源标识，用于路由到特定奖励函数 |

### Agent 训练特有字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `agent_name` | `str` | Agent 类型: `"tool_agent"` (多轮工具 Agent) 或 `"single_turn_agent"` (默认单轮) |
| `reward_model` | `dict` | 奖励配置: `{"style": "rule", "ground_truth": "..."}` |
| `extra_info` | `dict` | 扩展信息，包含 `tools_kwargs` 和 `interaction_kwargs` |

### 可选字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `ability` | `str` | 能力标签，如 `"math"`，用于分类和日志 |
| `images` | `list` | 图片数据列表，用于多模态场景（如几何问题） |

### extra_info 子字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `tools_kwargs` | `dict` | 工具参数，key = 工具名称，value = 包含 `create_kwargs` 等的 dict |
| `interaction_kwargs` | `dict` | 交互参数，必须包含 `"name"` 字段指定使用哪个交互 |
| `need_tools_kwargs` | `bool` | 标记是否需要传递 `tools_kwargs` |
| `question` | `str` | 原始问题文本 |
| `answer` | `str` | 原始答案文本 |
| `split` | `str` | 数据集划分标识（`"train"` / `"test"`） |
| `index` | `int` | 样本在原始数据集中的索引 |

### tools_kwargs 格式详解

`tools_kwargs` 用于为每个工具传入运行时参数，key 为工具名称（需与 `tool_config.yaml` 中定义的一致），value 为参数字典：

```python
"tools_kwargs": {
    "calc_gsm8k_reward": {           # 工具名称（需与 tool_config.yaml 中定义的一致）
        "create_kwargs": {            # 传给 tool.create() 的参数
            "ground_truth": "42"
        },
        "execute_kwargs": {},         # 可选：传给 tool.execute() 的额外参数
        "calc_reward_kwargs": {},     # 可选：传给 tool.calc_reward() 的额外参数
        "release_kwargs": {},         # 可选：传给 tool.release() 的额外参数
    }
}
```

这些参数对应 `BaseTool` 的生命周期方法（详见 [03 工具系统](./03_工具系统.md)）：
- `create_kwargs`: 工具初始化时传入的参数，如 ground truth 答案
- `execute_kwargs`: 工具执行时的额外参数
- `calc_reward_kwargs`: 计算奖励时的额外参数
- `release_kwargs`: 工具释放资源时的额外参数

### interaction_kwargs 格式详解

`interaction_kwargs` 用于配置交互系统的参数，`name` 字段指定交互名称（需与 `interaction_config.yaml` 中定义的一致）：

```python
"interaction_kwargs": {
    "name": "gsm8k",                 # 交互名称（需与 interaction_config.yaml 中定义的一致）
    "query": "原始问题",              # 传给 interaction.start_interaction()
    "ground_truth": "标准答案",       # 传给 interaction.start_interaction()
}
```

> **注意**：`tools_kwargs` 和 `interaction_kwargs` 并不互斥。在使用 Tool Agent + Interaction 的场景下，两者可以同时存在。例如 `gsm8k_tool_agent_loop.py` 同时配置了两者。

## 4. 数据预处理脚本解析

以 GSM8K Tool Agent 为例，源文件：`examples/data_preprocess/gsm8k_tool_agent_loop.py`

### 核心处理流程

```python
import argparse
import os
import re

import datasets

from verl.utils.hdfs_io import copy, makedirs


def extract_solution(solution_str):
    """从 GSM8K 答案字符串中提取 '#### 数字' 中的数字"""
    solution = re.search("#### (\\-?[0-9\\.\\,]+)", solution_str)
    assert solution is not None
    final_solution = solution.group(0)
    final_solution = final_solution.split("#### ")[1].replace(",", "")
    return final_solution


if __name__ == "__main__":
    parser = argparse.ArgumentParser()
    parser.add_argument("--local_dataset_path", default=None,
                        help="The local path to the raw dataset, if it exists.")
    parser.add_argument("--local_save_dir", default="~/data/gsm8k",
                        help="The save directory for the preprocessed dataset.")
    parser.add_argument("--hdfs_dir", default=None)

    args = parser.parse_args()
    data_source = "openai/gsm8k"

    # 1. 加载原始数据集
    if args.local_dataset_path is not None:
        dataset = datasets.load_dataset(args.local_dataset_path, "main")
    else:
        dataset = datasets.load_dataset(data_source, "main")

    train_dataset = dataset["train"]
    test_dataset = dataset["test"]

    instruction_following = "Let's think step by step and output the final answer after `####`."

    # 2. 定义数据处理函数
    def make_map_fn(split):
        def process_fn(example, idx):
            question_raw = example.pop("question")
            question = question_raw + " " + instruction_following

            answer_raw = example.pop("answer")
            solution = extract_solution(answer_raw)  # 提取 "#### 数字" 中的数字

            data = {
                "data_source": data_source,
                "agent_name": "tool_agent",                # 关键：指定使用 tool_agent
                "prompt": [
                    {
                        "role": "system",
                        "content": (
                            "You are a math expert. You are given a question and you "
                            "need to solve it step by step. ..."
                        ),
                    },
                    {"role": "user", "content": question},
                ],
                "ability": "math",
                "reward_model": {"style": "rule", "ground_truth": solution},
                "extra_info": {
                    "split": split,
                    "index": idx,
                    "answer": answer_raw,
                    "question": question_raw,
                    "need_tools_kwargs": True,
                    "tools_kwargs": {
                        "calc_gsm8k_reward": {
                            "create_kwargs": {"ground_truth": solution},
                        },
                    },
                    "interaction_kwargs": {
                        "query": question,
                        "ground_truth": solution,
                    },
                },
            }
            return data

        return process_fn

    # 3. 应用处理函数
    train_dataset = train_dataset.map(function=make_map_fn("train"), with_indices=True)
    test_dataset = test_dataset.map(function=make_map_fn("test"), with_indices=True)

    # 4. 保存为 Parquet
    train_dataset.to_parquet(os.path.join(args.local_save_dir, "train.parquet"))
    test_dataset.to_parquet(os.path.join(args.local_save_dir, "test.parquet"))
```

### 运行命令

```bash
python examples/data_preprocess/gsm8k_tool_agent_loop.py --local_save_dir ~/data/gsm8k
```

如果原始数据集已下载到本地：

```bash
python examples/data_preprocess/gsm8k_tool_agent_loop.py \
    --local_dataset_path /path/to/gsm8k \
    --local_save_dir ~/data/gsm8k
```

支持上传到 HDFS：

```bash
python examples/data_preprocess/gsm8k_tool_agent_loop.py \
    --local_save_dir ~/data/gsm8k \
    --hdfs_dir hdfs://path/to/gsm8k
```

## 5. System Prompt 设计要点

System prompt 对 Agent 训练效果至关重要，它决定了模型在 rollout 阶段的行为模式。

### GSM8K Tool Agent 示例

```python
system_prompt = (
    "You are a math expert. "
    "You are given a question and you need to solve it step by step. "
    "Reasoning step by step before any tool call. "
    "You should use the `calc_gsm8k_reward` tool after step by step solving "
    "the question, before generate final answer at least once and refine your "
    "answer if necessary. "
    "Put your final answer in the format of `#### <answer>`."
)
```

### GSM8K Interaction 模式示例

```python
system_prompt = (
    "You are a math expert. "
    "You are given a question and you need to solve it step by step. "
    "You should rethinking carefully if user point out your answer is wrong. "
    "Put your final answer in the format of `#### <answer>`."
)
```

### 设计原则

1. **明确角色定位**：告诉模型它的专业领域（如 "You are a math expert"）
2. **指示何时使用工具**：明确工具调用的时机和条件（如 "use the `calc_gsm8k_reward` tool after step by step solving"）
3. **指定输出格式**：便于奖励函数解析最终答案（如 `#### <answer>` 格式）
4. **鼓励分步推理**：引导 Chain-of-Thought 推理（如 "Reasoning step by step before any tool call"）
5. **允许修正**：鼓励模型根据工具反馈调整答案（如 "refine your answer if necessary"）

> **提示**：System prompt 的设计直接影响训练收敛速度和最终效果。建议在正式训练前，先用少量数据测试不同 prompt 的生成质量。

## 6. 多模态数据处理

当数据集包含图片或视频等多模态内容时（如几何问题），需要注意数据配置。

### 配置选项

```yaml
data:
  return_multi_modal_inputs: False  # Agent 训练设为 False，由 rollout 动态处理
```

- `return_multi_modal_inputs: True`（默认）：数据集在加载时预处理多模态输入，生成 `multi_modal_inputs` 字典（包含图片 tensor、视频 tensor 等）。适用于单轮训练或 SFT 场景。

- `return_multi_modal_inputs: False`：数据集不预处理多模态输入。适用于多轮 RL 或工具增强 rollout 场景，其中模型可能在 rollout 过程中动态产生新的多模态输入。

### 多模态数据示例

以 Geometry3K 为例（源文件：`examples/data_preprocess/geo3k_multiturn_w_tool.py`），数据中包含 `images` 字段：

```python
data = {
    "data_source": "hiyouga/geometry3k",
    "prompt": [
        {
            "role": "system",
            "content": "You are a math expert. ..."
        },
        {
            "role": "user",
            "content": problem + " " + instruction_following,
        },
    ],
    "images": images,                    # 图片数据
    "ability": "math",
    "reward_model": {"style": "rule", "ground_truth": answer},
    "extra_info": {
        "need_tools_kwargs": True,
        "tools_kwargs": {
            "calc_geo3k_reward": {
                "create_kwargs": {"ground_truth": answer},
            },
        },
    },
}
```

## 7. 自定义数据集准备步骤

按照以下步骤构建自己的 Agent 训练数据集：

### 步骤概览

1. **准备原始数据**：收集问题和标准答案
2. **设计 System Prompt**：指导模型使用工具和输出格式
3. **构造 prompt 字段**：chat format 消息列表
4. **设置 agent_name**：`"tool_agent"` 或自定义 Agent 名称
5. **配置 tools_kwargs**：为每个工具传入所需参数
6. **配置 interaction_kwargs**：如使用交互系统
7. **设置 reward_model**：奖励计算所需的 ground truth
8. **保存为 Parquet**：使用 `datasets` 库的 `to_parquet()`

### 完整代码模板

```python
import datasets

# 假设已有以下数据
questions = ["What is 2+3?", "What is 10*5?"]
ground_truths = ["5", "50"]
sys_prompt = "You are a math expert. Use the `my_tool` tool to verify your answer."

# 构造数据字典
data = {
    "data_source": ["my_task"] * len(questions),
    "agent_name": ["tool_agent"] * len(questions),
    "prompt": [
        [
            {"role": "system", "content": sys_prompt},
            {"role": "user", "content": q},
        ]
        for q in questions
    ],
    "reward_model": [
        {"style": "rule", "ground_truth": gt}
        for gt in ground_truths
    ],
    "extra_info": [
        {
            "need_tools_kwargs": True,
            "tools_kwargs": {
                "my_tool": {
                    "create_kwargs": {"ground_truth": gt},
                }
            },
            "interaction_kwargs": {
                "name": "my_interaction",
                "ground_truth": gt,
            },
        }
        for gt in ground_truths
    ],
}

# 创建 Dataset 并保存
dataset = datasets.Dataset.from_dict(data)
dataset.to_parquet("my_data/train.parquet")
```

### 验证数据格式

保存后可以用以下代码验证数据是否正确：

```python
import datasets

# 重新加载并检查
ds = datasets.load_dataset("parquet", data_files="my_data/train.parquet")["train"]
print(f"样本数量: {len(ds)}")
print(f"字段列表: {ds.column_names}")
print(f"第一条样本: {ds[0]}")

# 检查关键字段是否存在
sample = ds[0]
assert "prompt" in sample, "缺少 prompt 字段"
assert "data_source" in sample, "缺少 data_source 字段"
assert isinstance(sample["prompt"], list), "prompt 应为 list 类型"
assert all("role" in msg and "content" in msg for msg in sample["prompt"]), \
    "prompt 中每条消息需包含 role 和 content"
```

verl 提供了多个预处理脚本，覆盖不同任务类型和 Agent 模式：

| 脚本 | 任务 | 特点 | 路径 |
|------|------|------|------|
| `gsm8k_tool_agent_loop.py` | GSM8K 数学 + 工具 Agent | 使用 `agent_name="tool_agent"`，配置 `tools_kwargs` + `interaction_kwargs` | `examples/data_preprocess/` |
| `gsm8k_multiturn_w_tool.py` | GSM8K 多轮 + 工具 | 配置 `tools_kwargs`，无 `agent_name` 字段 | `examples/data_preprocess/` |
| `gsm8k_multiturn_w_interaction.py` | GSM8K 多轮 + 交互 | 仅配置 `interaction_kwargs`（含 `name` 字段），无工具参数 | `examples/data_preprocess/` |
| `aime2024_multiturn_w_tool.py` | AIME 2024 竞赛数学 | 使用 `code_interpreter` 工具，基于已有数据集追加 `tools_kwargs` | `examples/data_preprocess/` |
| `geo3k_multiturn_w_tool.py` | 几何问题 + 多模态 | 包含 `images` 字段，使用 `calc_geo3k_reward` 工具 | `examples/data_preprocess/` |
| `dapo_multiturn_w_tool.py` | DAPO 训练数据 | 使用 `code_interpreter` 工具，基于 DAPO-Math-17k 数据集 | `examples/data_preprocess/` |

### 各脚本使用的数据源

| 脚本 | HuggingFace 数据源 |
|------|-------------------|
| `gsm8k_tool_agent_loop.py` | `openai/gsm8k` |
| `gsm8k_multiturn_w_tool.py` | `openai/gsm8k` |
| `gsm8k_multiturn_w_interaction.py` | `openai/gsm8k` |

### 关键差异对比

不同预处理脚本之间的关键差异在于是否使用工具、是否使用交互系统以及是否指定 `agent_name`：

| 模式 | `agent_name` | `tools_kwargs` | `interaction_kwargs` | 适用场景 |
|------|-------------|---------------|---------------------|---------|
| Tool Agent | `"tool_agent"` | 必需 | 可选 | 模型主动调用工具，多轮推理 |
| Interaction Only | 无 | 无 | 必需（含 `name`） | 环境驱动的多轮交互 |
| Tool + Interaction | `"tool_agent"` | 必需 | 必需 | 工具调用 + 环境交互结合 |

## 9. 常见问题

### Q: `tools_kwargs` 中的工具名称从哪里来？

工具名称需要与 `tool_config.yaml` 中定义的工具名称一致。例如 `calc_gsm8k_reward` 对应工具配置中注册的同名工具。详见 [03 工具系统](./03_工具系统.md)。

### Q: `interaction_kwargs` 中的 `name` 字段是什么？

`name` 字段指定使用哪个交互实现，需要与 `interaction_config.yaml` 中注册的交互名称一致。例如 `"gsm8k"` 对应 GSM8K 交互实现。详见 [04 交互系统](./04_交互系统.md)。

### Q: 能否不使用 `extra_info`，直接在顶层放置 `tools_kwargs`？

不推荐。verl 的数据加载管线会将 `extra_info` 中的内容传递到 `DataProto` 的 `non_tensor_batch` 中，Agent Loop 从中读取工具和交互参数。将这些字段放在 `extra_info` 中是约定的数据协议。

### Q: `need_tools_kwargs` 是否必需？

当使用 `tools_kwargs` 时建议设置 `need_tools_kwargs: True`，这个标记帮助 Agent Loop 判断是否需要从数据中提取工具参数。如果未设置，某些工具可能无法正确接收到初始化参数。

### Q: 如何处理大规模数据集？

对于大规模数据集，可以利用 `datasets` 库的并行处理能力：

```python
# 使用多进程加速数据处理
train_dataset = train_dataset.map(
    function=make_map_fn("train"),
    with_indices=True,
    num_proc=8,              # 8 个进程并行处理
)
```

此外，Parquet 格式本身支持分片存储，可以将大数据集拆分为多个文件：

```yaml
data:
  train_files:
    - ~/data/my_task/train_part1.parquet
    - ~/data/my_task/train_part2.parquet
```
