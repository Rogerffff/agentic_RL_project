# verl Agent 训练文档

verl 是字节跳动 Seed 团队发起的开源 RL 后训练框架，实现了 HybridFlow 架构（EuroSys 2025），支持 PPO、GRPO、DAPO 等多种 RL 算法。Agent 训练功能使模型能在多轮对话中使用工具、与环境交互并通过强化学习优化。

本文档系列系统介绍 verl 中 Agent 训练的架构设计、核心组件、配置方法与实战流程，帮助开发者快速理解和上手 Agent RL 训练。

## 核心概念速查表

| 概念 | 说明 | 详见 |
|------|------|------|
| AgentLoop | 多轮 Agent 执行循环，管理 LLM 生成 → 工具调用 → 交互反馈的完整流程 | [02 AgentLoop 详解](./02_AgentLoop详解.md) |
| ToolAgentLoop | 内置的工具 Agent 实现，基于状态机驱动的多轮工具调用 | [02 AgentLoop 详解](./02_AgentLoop详解.md) |
| AsyncLLMServerManager | 推理网关，提供负载均衡和粘性会话的 LLM 生成接口 | [01 架构设计](./01_架构设计.md) |
| DataProto | 通用数据交换协议，所有组件间通信的标准格式 | [01 架构设计](./01_架构设计.md) |
| response_mask | 响应掩码，区分 LLM 生成 token (1) 和工具响应 token (0) | [02 AgentLoop 详解](./02_AgentLoop详解.md) |
| BaseTool | 工具抽象基类，定义 `create`/`execute`/`calc_reward`/`release` 生命周期 | [03 工具系统](./03_工具系统.md) |
| BaseInteraction | 交互抽象基类，支持多轮环境交互和轮次级评分 | [04 交互系统](./04_交互系统.md) |
| RewardLoop | 分布式奖励计算框架，支持规则奖励、判别式/生成式奖励模型 | [05 奖励系统](./05_奖励系统.md) |
| GRPO | Group Relative Policy Optimization，适用于 Agent 训练的优势估计算法 | [06 训练流程](./06_训练流程.md) |
