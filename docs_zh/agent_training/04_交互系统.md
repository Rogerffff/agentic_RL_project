# 交互系统 (Interaction System)

## 1. 交互系统概述

verl 的交互系统 (Interaction System) 使模型能够在 RL 训练期间进行动态的多轮对话反馈。交互代理 (Interaction Agent) 可以根据模型的响应提供纠正反馈、指导或评估，实现迭代式的问题解决训练场景。

与工具系统 (Tool System) 不同，交互系统关注的是**模型与环境/评估者之间的多轮对话**，而非单次的函数调用。典型应用场景包括：模型给出一个数学答案后，交互代理判断正确性并给出反馈，模型据此反思并重新作答。

**关键特性：**

- **基于 asyncio 的非阻塞架构**：交互处理与分布式训练流程无缝衔接
- **有状态的会话管理**：通过唯一 instance ID 管理并发交互会话
- **配置驱动的动态加载**：通过 YAML 配置文件动态加载交互类，无需修改核心代码
- **多交互注册与样本级选择**：同一训练中可注册多个交互代理，每个样本通过 `name` 字段选择使用哪个交互
- **轮次级评分机制**：每轮交互产生 `turn_score`，与 verl 的奖励系统集成

## 2. BaseInteraction 接口设计

> 源文件：`verl/interactions/base.py`

`BaseInteraction` 是所有交互代理的抽象基类，定义了交互的完整生命周期接口：

```python
from typing import Any, Optional
from uuid import uuid4


class BaseInteraction:
    def __init__(self, config: dict[str, Any]):
        self.config = config
        self.name: str = config.get("name", "interaction_agent")

    async def start_interaction(self, instance_id: Optional[str] = None, **kwargs) -> str:
        """初始化交互会话。

        Args:
            instance_id: 交互实例 ID，若为 None 则自动生成。

        Returns:
            交互实例的唯一 ID。
        """
        if instance_id is None:
            return str(uuid4())
        else:
            return instance_id

    async def generate_response(
        self, instance_id: str, messages: list[dict[str, Any]], **kwargs
    ) -> tuple[bool, str, float, dict[str, Any]]:
        """生成交互响应，评估模型的当前输出。

        Args:
            instance_id: 交互实例 ID。
            messages: 当前对话的完整消息列表。

        Returns:
            一个四元组：
            - should_terminate_sequence (bool): 若为 True，则结束当前 rollout
            - response_content (str): 交互反馈文本，将作为 user 消息追加到对话中
            - current_turn_score (float): 本轮评分，用于奖励计算
            - additional_data (dict): 额外元数据
        """
        should_terminate_sequence: bool = False
        response_content: str = "Your current result seems acceptable."
        current_turn_score: float = 0.8
        additional_data: dict[str, Any] = {}
        return should_terminate_sequence, response_content, current_turn_score, additional_data

    async def calculate_score(self) -> float:
        """计算轮次级评分。

        用于支持 partial exposure 和 in-context task switching 等高级评分逻辑。
        应在每轮交互时调用。
        """
        score = 0.0
        return score

    async def finalize_interaction(self) -> None:
        """终结交互会话，释放关联的状态和资源。"""
        pass
```

### 交互生命周期

一个完整的交互会话遵循以下生命周期流程：

```
start_interaction()        generate_response()        generate_response()          finalize_interaction()
       |                          |                          |                             |
       v                          v                          v                             v
  初始化会话状态            评估模型响应,              继续评估,                      清理资源,
  接收 ground_truth        返回反馈和评分            直到回答正确或达到上限          释放 instance 状态
  返回 instance_id         (should_terminate=False)  (should_terminate=True)
```

### 返回值语义

`generate_response()` 的四元组返回值是交互系统的核心设计：

| 返回值 | 类型 | 说明 |
|--------|------|------|
| `should_terminate_sequence` | `bool` | `True` 表示终止 rollout，模型不再继续生成；`False` 表示继续下一轮对话 |
| `response_content` | `str` | 反馈文本，作为 `role: user` 消息追加到对话历史，供模型在下一轮生成时参考 |
| `current_turn_score` | `float` | 本轮评分，存入 `AgentData.turn_scores` 列表，用于奖励计算 |
| `additional_data` | `dict` | 额外元数据，可用于 logging 或自定义逻辑 |

## 3. 交互在 Agent 循环中的工作流程

交互系统通过 `ToolAgentLoop` 中的状态机与 Agent 循环集成。

> 源文件：`verl/experimental/agent_loop/tool_agent_loop.py`

### 状态机设计

`ToolAgentLoop` 使用 `AgentState` 枚举管理状态转换：

```python
class AgentState(Enum):
    PENDING = "pending"            # 初始状态，准备 prompt
    GENERATING = "generating"      # 模型生成响应
    PROCESSING_TOOLS = "processing_tools"  # 处理工具调用
    TERMINATED = "terminated"      # 终止
    INTERACTING = "interacting"    # 交互代理评估响应
```

### 完整状态转换流程

```
                    +---------------------------------------------+
                    |            ToolAgentLoop                     |
                    |                                             |
  PENDING -----> GENERATING -----> 检查模型输出 ------+
                    ^                                  |
                    |                     +------------+------------+
                    |                     |            |            |
                    |                有 tool_calls   有 interaction   无两者
                    |                     |            |            |
                    |                     v            v            v
                    |            PROCESSING_TOOLS  INTERACTING  TERMINATED
                    |                     |            |
                    |                     |    generate_response()
                    |                     |            |
                    |                     |    +-------+--------+
                    |                     |    |                |
                    |                     | terminate=False  terminate=True
                    |                     |    |                |
                    +---------------------+----+           TERMINATED
                         (继续下一轮)
```

### 关键逻辑：`_handle_generating_state`

在模型生成完成后，`_handle_generating_state` 方法决定下一个状态：

```python
# 提取 tool calls
_, agent_data.tool_calls = await self.tool_parser.extract_tool_calls(agent_data.response_ids)

# 如果配置了交互，将 assistant 消息追加到对话历史
if self.interaction_config_file:
    assistant_message = await self.loop.run_in_executor(
        None, lambda: self.tokenizer.decode(agent_data.response_ids, skip_special_tokens=True)
    )
    add_messages.append({"role": "assistant", "content": assistant_message})
    agent_data.messages.extend(add_messages)

# 状态转换逻辑
if agent_data.tool_calls:
    return AgentState.PROCESSING_TOOLS     # 优先处理工具调用
elif self.interaction_config_file:
    return AgentState.INTERACTING           # 无工具调用时进入交互状态
else:
    return AgentState.TERMINATED            # 无交互配置则终止
```

### 关键逻辑：`_handle_interacting_state`

交互状态的处理逻辑：

```python
async def _handle_interacting_state(self, agent_data: AgentData) -> AgentState:
    # 调用交互代理的 generate_response
    (
        should_terminate_sequence,
        interaction_responses,
        reward,
        metrics,
    ) = await agent_data.interaction.generate_response(
        agent_data.request_id, agent_data.messages, **agent_data.interaction_kwargs
    )
    agent_data.user_turns += 1

    # 将交互反馈作为 user 消息追加
    add_messages = [{"role": "user", "content": interaction_responses}]
    agent_data.messages.extend(add_messages)

    # 记录轮次评分
    if reward is not None:
        agent_data.turn_scores.append(reward)

    # 对反馈文本进行 tokenize，更新 prompt_ids 和 response_mask
    response_ids = await self.apply_chat_template(add_messages, remove_system_prompt=True)
    agent_data.prompt_ids += response_ids
    agent_data.response_mask += [0] * len(response_ids)  # 交互反馈 token 的 mask 为 0
    if agent_data.response_logprobs:
        agent_data.response_logprobs += [0.0] * len(response_ids)

    # 根据交互返回值决定是否终止
    if should_terminate_sequence:
        return AgentState.TERMINATED
    else:
        return AgentState.GENERATING  # 继续下一轮模型生成
```

**要点**：交互反馈 token 的 `response_mask` 设为 `0`，这意味着这些 token 不参与 policy loss 计算 -- 只有模型自身生成的 token 才被训练。

### AgentData 中的交互相关字段

```python
class AgentData:
    def __init__(self, ..., interaction=None, interaction_kwargs=None):
        self.interaction = interaction              # BaseInteraction 实例
        self.interaction_kwargs = interaction_kwargs  # 传递给交互的参数
        self.turn_scores: list[float] = []          # 每轮交互的评分列表
        self.tool_rewards: list[float] = []         # 每次工具调用的奖励列表
        # ...
```

`turn_scores` 和 `tool_rewards` 最终通过 `AgentLoopOutput.extra_fields` 返回：

```python
output.extra_fields.update({
    "turn_scores": agent_data.turn_scores,
    "tool_rewards": agent_data.tool_rewards,
})
```

## 5. 多交互注册与样本级选择

> 源文件：`verl/interactions/utils/interaction_registry.py`

### 注册机制

系统支持在同一训练中注册多个交互代理。`initialize_interactions_from_config()` 函数从配置文件加载所有交互，返回一个 `name -> BaseInteraction` 的字典：

```python
from verl.interactions.utils.interaction_registry import initialize_interactions_from_config

# 加载配置并初始化所有交互
interaction_map = initialize_interactions_from_config("interaction_config.yaml")
# interaction_map = {"gsm8k": Gsm8kInteraction(...), "code_verifier": CodeVerifier(...)}
```

内部实现使用 `importlib` 动态导入指定的类：

```python
def get_interaction_class(cls_name):
    """动态导入交互类。"""
    module_name, class_name = cls_name.rsplit(".", 1)
    if module_name not in sys.modules:
        spec = importlib.util.find_spec(module_name)
        module = importlib.util.module_from_spec(spec)
        sys.modules[module_name] = module
        spec.loader.exec_module(module)
    else:
        module = sys.modules[module_name]
    return getattr(module, class_name)
```

### 样本级选择

每个样本通过 `extra_info.interaction_kwargs` 中的 `name` 字段指定使用哪个交互代理。在 `ToolAgentLoop.run()` 中，系统根据 `name` 从 `interaction_map` 中查找对应的交互实例：

```python
# ToolAgentLoop.run() 中的交互初始化逻辑
if self.interaction_config_file:
    interaction_kwargs = kwargs["extra_info"]["interaction_kwargs"]
    if "name" not in interaction_kwargs:
        raise ValueError("'name' key is required in interaction_kwargs")
    interaction_name = interaction_kwargs["name"]
    if interaction_name not in self.interaction_map:
        raise ValueError(
            f"Interaction '{interaction_name}' not found in interaction_map. "
            f"Available interactions: {list(self.interaction_map.keys())}"
        )
    interaction = self.interaction_map[interaction_name]
    await interaction.start_interaction(request_id, **interaction_kwargs)
```

**注意**：当配置了 `interaction_config_path` 时，`name` 字段是**必需**的。如果 `interaction_kwargs` 中缺少 `name`，系统会抛出 `ValueError`。

## 6. GSM8K 交互实现示例

> 源文件：`verl/interactions/gsm8k_interaction.py`

`Gsm8kInteraction` 是一个完整的交互实现示例，用于数学问题求解场景中的迭代式训练：

```python
import logging
import os
from typing import Any, Optional
from uuid import uuid4

from verl.utils.reward_score import gsm8k

from .base import BaseInteraction

logger = logging.getLogger(__name__)
logger.setLevel(os.getenv("VERL_LOGGING_LEVEL", "WARN"))


class Gsm8kInteraction(BaseInteraction):
    def __init__(self, config: dict):
        super().__init__(config)
        self._instance_dict = {}          # instance_id -> 会话状态

    async def start_interaction(
        self, instance_id: Optional[str] = None, ground_truth: Optional[str] = None, **kwargs
    ) -> str:
        if instance_id is None:
            instance_id = str(uuid4())
        self._instance_dict[instance_id] = {
            "response": "",
            "ground_truth": ground_truth,  # 存储正确答案
            "reward": 0.0,
        }
        return instance_id

    async def generate_response(
        self, instance_id: str, messages: list[dict[str, Any]], **kwargs
    ) -> tuple[bool, str, float, dict]:
        # 从对话历史中提取最后一条 assistant 消息
        content = ""
        for i in range(len(messages) - 1, -1, -1):
            item = messages[i]
            if item.get("role") == "assistant":
                content = item.get("content")
                break

        self._instance_dict[instance_id]["response"] = content

        # 计算评分
        reward = await self.calculate_score(instance_id)
        if reward == 1.0:
            response = "Your response is correct!"
            should_terminate_sequence = True   # 答对 -> 终止
        else:
            response = "Your response is incorrect! You need to reflect on your answer and try again."
            should_terminate_sequence = False   # 答错 -> 继续

        return should_terminate_sequence, response, reward, {}

    async def calculate_score(self, instance_id: str, **kwargs) -> float:
        return gsm8k.compute_score(
            self._instance_dict[instance_id]["response"],
            self._instance_dict[instance_id]["ground_truth"],
            method="strict",
            format_score=0.0,
            score=1.0,
        )

    async def finalize_interaction(self, instance_id: str, **kwargs) -> None:
        del self._instance_dict[instance_id]   # 清理会话状态
```

### 核心逻辑

| 场景 | `should_terminate` | `reward` | 反馈内容 |
|------|-------------------|----------|----------|
| 模型答对 | `True` | `1.0` | "Your response is correct!" |
| 模型答错 | `False` | `0.0` | "Your response is incorrect! You need to reflect on your answer and try again." |

答错时返回 `should_terminate=False`，模型会收到反馈消息并继续生成新的回答，直到答对或达到 `max_user_turns` / `max_assistant_turns` 上限。

### Weather 交互示例

> 源文件：`verl/interactions/weather_interaction.py`

verl 还提供了一个 `WeatherInteraction` 示例，展示了与工具系统结合的交互模式：

```python
class WeatherInteraction(BaseInteraction):
    async def generate_response(self, instance_id, messages, **kwargs):
        # 检查最后一条 tool 消息（而非 assistant 消息）
        content = "no tool call"
        for i in range(len(messages) - 1, -1, -1):
            item = messages[i]
            if item.get("role") == "tool":
                content = item.get("content")
                break
        self._instance_dict[instance_id]["response"] = content

        reward = await self.calculate_score(instance_id)
        if reward == 1.0:
            response = "Thank you for your weather query!"
        else:
            response = "Please use the weather tool to get the weather information."
        # 注意：两种情况都返回 should_terminate=True
        should_terminate_sequence = True
        return should_terminate_sequence, response, reward, {}
```

这个例子展示了交互代理如何检查模型是否正确使用了工具。

## 7. Curriculum Learning 高级用法

利用多交互支持，可以实现课程学习 (Curriculum Learning) 策略，在训练的不同阶段使用不同难度的交互代理：

### 交互配置

```yaml
# curriculum_interaction_config.yaml
interaction:
  - name: "basic_math"
    class_name: "verl.interactions.gsm8k_interaction.Gsm8kInteraction"
    config:
      difficulty: "easy"

  - name: "advanced_math"
    class_name: "custom.interactions.AdvancedMathInteraction"
    config:
      difficulty: "hard"
      allow_hints: true

  - name: "competition_math"
    class_name: "custom.interactions.CompetitionMathInteraction"
    config:
      time_limit: 300
      show_steps: false
```

### 数据集构造

```python
class MathTrainingPipeline:
    def create_curriculum_data(self, epoch, samples):
        """根据训练 epoch 动态调整样本使用的交互代理。"""
        if epoch < 5:
            # 早期：简单数学题，使用基础交互
            return [
                {
                    "extra_info": {
                        "interaction_kwargs": {
                            "name": "basic_math",
                            "ground_truth": sample["answer"],
                        }
                    }
                }
                for sample in samples
            ]
        elif epoch < 10:
            # 中期：进阶数学，允许提示
            return [
                {
                    "extra_info": {
                        "interaction_kwargs": {
                            "name": "advanced_math",
                            "ground_truth": sample["answer"],
                        }
                    }
                }
                for sample in samples
            ]
        else:
            # 后期：竞赛级数学，不给提示
            return [
                {
                    "extra_info": {
                        "interaction_kwargs": {
                            "name": "competition_math",
                            "ground_truth": sample["answer"],
                        }
                    }
                }
                for sample in samples
            ]
```

这种模式的优势在于：交互配置是静态的（训练启动时全部加载），而数据集中的 `name` 字段可以动态变化，灵活控制每个样本使用哪个交互策略。

## 8. 自定义交互开发指南

### Step 1：继承 BaseInteraction

```python
from typing import Any, Optional
from uuid import uuid4

from verl.interactions.base import BaseInteraction


class MyInteraction(BaseInteraction):
    def __init__(self, config: dict[str, Any]):
        super().__init__(config)
        self._instances: dict[str, dict] = {}

    async def start_interaction(
        self, instance_id: Optional[str] = None, **kwargs
    ) -> str:
        instance_id = instance_id or str(uuid4())
        self._instances[instance_id] = {
            "ground_truth": kwargs.get("ground_truth"),
            "attempts": 0,
            "max_attempts": self.config.get("max_attempts", 3),
        }
        return instance_id

    async def generate_response(
        self, instance_id: str, messages: list[dict[str, Any]], **kwargs
    ) -> tuple[bool, str, float, dict[str, Any]]:
        instance = self._instances[instance_id]
        instance["attempts"] += 1

        # 提取模型最后的响应
        last_response = ""
        for item in reversed(messages):
            if item.get("role") == "assistant":
                last_response = item.get("content", "")
                break

        # 评估响应
        is_correct = self._evaluate(last_response, instance["ground_truth"])

        if is_correct:
            return True, "Correct! Well done.", 1.0, {"attempts": instance["attempts"]}
        elif instance["attempts"] >= instance["max_attempts"]:
            return True, "Max attempts reached.", 0.0, {"attempts": instance["attempts"]}
        else:
            hint = self._generate_hint(instance["attempts"])
            return False, f"Incorrect. {hint}", 0.0, {"attempts": instance["attempts"]}

    async def calculate_score(self, instance_id: str, **kwargs) -> float:
        # 可选：实现更复杂的评分逻辑
        return 0.0

    async def finalize_interaction(self, instance_id: str, **kwargs) -> None:
        del self._instances[instance_id]

    def _evaluate(self, response: str, ground_truth: str) -> bool:
        """自定义评估逻辑。"""
        # 实现你的评估方法
        return response.strip() == ground_truth.strip()

    def _generate_hint(self, attempt: int) -> str:
        """根据尝试次数生成递进式提示。"""
        if attempt == 1:
            return "Try again with a different approach."
        else:
            return "Think more carefully about the problem."
```

## 9. 最佳实践

### 资源管理

- **始终在 `finalize_interaction()` 中清理资源**：使用 `del self._instance_dict[instance_id]` 释放会话状态，避免内存泄漏
- **使用唯一 instance_id**：`ToolAgentLoop` 默认使用 `uuid4().hex` 作为 `request_id`，确保并发安全
- **处理边界情况**：在 `generate_response()` 中处理空消息列表、缺失 assistant 消息等异常情况

### 性能优化

- **保持交互逻辑轻量**：`generate_response()` 在 Agent 循环的关键路径上，应避免耗时操作
- **正确使用 async/await**：所有交互方法都是 async 的，确保不会阻塞事件循环
- **缓存重复计算**：如果评分逻辑涉及重复计算，可在 `_instance_dict` 中缓存中间结果

### 评分设计

- **turn_score 与 reward 的关系**：`turn_scores` 列表记录每轮交互的评分，可用于在 reward manager 中实现多轮奖励聚合
- **二值 vs. 连续评分**：GSM8K 示例使用二值评分（0.0 或 1.0），也可以使用连续评分来提供更细粒度的奖励信号
- **终止条件设计**：合理设置 `should_terminate`，既要给模型足够的尝试机会，又要避免无效的循环

### 调试

启用 debug 日志以追踪交互流程：

```bash
export VERL_LOGGING_LEVEL=DEBUG
```

交互系统的日志使用 `logging` 模块，日志级别通过 `VERL_LOGGING_LEVEL` 环境变量控制（默认 `WARN`）。

### 测试

为自定义交互编写异步测试：

```python
import pytest


@pytest.mark.asyncio
async def test_interaction_workflow():
    from my_module.interactions import MyInteraction

    interaction = MyInteraction({"name": "test", "max_attempts": 3})

    # 测试完整工作流
    instance_id = await interaction.start_interaction(ground_truth="42")

    # 第一轮：错误答案
    messages = [{"role": "assistant", "content": "The answer is 40."}]
    should_terminate, response, reward, metadata = await interaction.generate_response(instance_id, messages)

    assert reward == 0.0

    messages.append({"role": "assistant", "content": "The answer is 42."})
    should_terminate, response, reward, metadata = await interaction.generate_response(instance_id, messages)
    assert should_terminate is True
    assert reward == 1.0

    # 清理
    await interaction.finalize_interaction(instance_id)
```

## 10. 与工具系统的对比

交互系统和工具系统都是 Agent 循环的核心组件，但它们服务于不同的目的：

| 维度 | 工具系统 (Tool System) | 交互系统 (Interaction System) |
|------|----------------------|------------------------------|
| 触发条件 | 模型输出包含 tool calls | 模型输出不包含 tool calls |
| 消息角色 | `role: tool` | `role: user` |
| response_mask | `0`（不参与 loss 计算） | `0`（不参与 loss 计算） |
| 主要用途 | 执行外部操作（搜索、计算、代码执行） | 评估模型响应并提供反馈 |
| 状态管理 | `create` / `execute` / `release` | `start_interaction` / `generate_response` / `finalize_interaction` |
| 评分方式 | `tool_rewards` | `turn_scores` |
| 典型场景 | 计算器、搜索引擎、代码沙箱 | 数学答案验证、多轮纠错、对话质量评估 |

在 `ToolAgentLoop` 的状态转换中，工具调用 (`PROCESSING_TOOLS`) 的优先级高于交互 (`INTERACTING`)。如果模型同时生成了 tool calls，系统会先处理工具调用，而不是进入交互状态。
