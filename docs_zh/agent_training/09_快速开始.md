# 快速开始：Agent 训练端到端指南

本文档提供一个完整的端到端快速上手指南，帮助你从零开始在 verl 中完成一次 Agent 训练。我们以 GSM8K Tool Agent 为例，展示模型如何学习在多轮对话中使用计算器工具验证数学答案。

---

## 1. 环境准备

### 1.1 安装 verl

```bash
# 克隆仓库
git clone https://github.com/volcengine/verl.git
cd verl

# 使用 SGLang 后端安装（推荐用于 Agent 训练）
pip install -e ".[test,sglang]"

# 或使用 vLLM 后端
pip install -e ".[test,vllm]"

# GPU 优化（可选）
pip install -e ".[gpu]"  # flash-attn + liger-kernel
```

### 1.2 安装追踪工具（推荐）

MLflow 用于记录 Agent 训练过程中的完整对话历史、工具调用参数和返回结果，强烈建议安装：

```bash
pip install mlflow
```

### 1.3 硬件要求

| 规模 | GPU 配置 | 说明 |
|------|---------|------|
| 最低 | 2 x GPU（例如 2 x A100/H100） | 需要调整 batch size 和并行参数 |
| 推荐 | 8 x GPU（例如 8 x H100） | 默认配置直接可用 |
| 显存 | 每 GPU 至少 40GB | 多轮对话会消耗更多 KV Cache 显存 |

## 2. GSM8K Tool Agent 端到端示例

这是一个完整的 GSM8K 数学问题 Agent 训练示例。模型学习使用计算器工具 `calc_gsm8k_reward` 验证自己的答案，根据工具返回的正确/错误反馈来优化解题策略。

### 2.1 数据准备

```bash
# 下载并预处理 GSM8K 数据集，添加 agent_name 字段
python examples/data_preprocess/gsm8k_tool_agent_loop.py --local_save_dir ~/data/gsm8k
```

这会生成：

- `~/data/gsm8k/train.parquet` — 训练集（7473 条）
- `~/data/gsm8k/test.parquet` — 测试集（1319 条）

每条数据包含以下关键字段：

| 字段 | 说明 |
|------|------|
| `agent_name="tool_agent"` | 指定使用 `ToolAgentLoop` 进行多轮工具 Agent 推理 |
| `prompt` | 包含 system prompt（指导模型使用 `calc_gsm8k_reward` 工具）和 user 问题 |
| `extra_info.tools_kwargs` | 工具运行时所需的参数，包含 `ground_truth`（正确答案） |
| `reward_model` | 奖励模型配置，使用 rule-based 规则奖励 |

数据预处理脚本的核心逻辑（参见 `examples/data_preprocess/gsm8k_tool_agent_loop.py`）：

```python
data = {
    "data_source": "openai/gsm8k",
    "agent_name": "tool_agent",           # 使用多轮工具 Agent
    "prompt": [
        {"role": "system", "content": "You are a math expert..."},
        {"role": "user", "content": question},
    ],
    "reward_model": {"style": "rule", "ground_truth": solution},
    "extra_info": {
        "need_tools_kwargs": True,
        "tools_kwargs": {
            "calc_gsm8k_reward": {        # 工具名称，需与 tool_config 一致
                "create_kwargs": {"ground_truth": solution},
            },
        },
    },
}
```

### 2.2 理解工具配置

工具配置文件位于 `examples/sglang_multiturn/config/tool_config/gsm8k_tool_config.yaml`：

```yaml
tools:
  - class_name: "verl.tools.gsm8k_tool.Gsm8kTool"
    config:
      type: native
    tool_schema:
      type: "function"
      function:
        name: "calc_gsm8k_reward"
        description: "A tool for calculating the reward of gsm8k. (1.0 if parsed answer is correct, 0.0 if parsed answer is incorrect or not correctly parsed)"
        parameters:
          type: "object"
          properties:
            answer:
              type: "string"
              description: "The model's answer to the GSM8K math problem, must be a digits"
          required: ["answer"]
```

工具的工作流程：

1. 模型在推理过程中生成工具调用，传入自己计算得到的 `answer`
2. `Gsm8kTool` 将模型的 `answer` 与数据中的 `ground_truth` 进行比较
3. 返回奖励值：`1.0`（正确）或 `0.0`（错误或解析失败）
4. 模型根据工具反馈决定是否调整答案

### 2.3 理解训练配置

训练配置文件位于 `examples/sglang_multiturn/config/gsm8k_multiturn_grpo.yaml`：

```yaml
hydra:
  searchpath:
    - file://verl/trainer/config

defaults:
  - ppo_trainer
  - _self_

data:
  max_prompt_length: 1024
  max_response_length: 1024
  train_batch_size: 256
  return_raw_chat: True          # 必须: 启用原始聊天格式，Agent 循环所需

actor_rollout_ref:
  hybrid_engine: True
  rollout:
    name: sglang                  # 使用 SGLang 推理引擎
    multi_turn:
      enable: True                # 必须: 启用多轮对话
      max_assistant_turns: 5      # 最多 5 轮 assistant 回复
```

关键配置说明：

- `return_raw_chat: True` — 保留原始聊天格式（`List[Dict]`），Agent Loop 需要聊天历史来管理多轮对话
- `rollout.name: sglang` — 使用 SGLang 作为推理引擎，支持异步多轮推理
- `multi_turn.enable: True` — 激活多轮推理模式，启用 Agent Loop
- `multi_turn.max_assistant_turns: 5` — 限制最大 assistant 回复轮数，避免对话过长导致显存溢出

### 2.4 启动训练

**方式一：使用现成的训练脚本（推荐）**

```bash
# 确保在项目根目录下
cd /path/to/verl

# 运行训练脚本（8 x GPU）
bash examples/sglang_multiturn/run_qwen2.5-3b_gsm8k_tool_agent_mlflow.sh
```

**方式二：手动运行完整命令**

```bash
PROJECT_DIR="$(pwd)"
CONFIG_PATH="$PROJECT_DIR/examples/sglang_multiturn/config"

python3 -m verl.trainer.main_ppo \
    --config-path="$CONFIG_PATH" \
    --config-name='gsm8k_multiturn_grpo' \
    algorithm.adv_estimator=grpo \
    data.train_batch_size=256 \
    data.max_prompt_length=1024 \
    data.max_response_length=1024 \
    data.filter_overlong_prompts=True \
    data.truncation='error' \
    data.return_raw_chat=True \
    actor_rollout_ref.model.path=Qwen/Qwen2.5-3B-Instruct \
    actor_rollout_ref.actor.optim.lr=1e-6 \
    actor_rollout_ref.model.use_remove_padding=True \
    actor_rollout_ref.actor.ppo_mini_batch_size=256 \
    actor_rollout_ref.actor.ppo_micro_batch_size_per_gpu=32 \
    actor_rollout_ref.actor.use_kl_loss=True \
    actor_rollout_ref.actor.kl_loss_coef=0.001 \
    actor_rollout_ref.actor.kl_loss_type=low_var_kl \
    actor_rollout_ref.actor.entropy_coeff=0 \
    actor_rollout_ref.model.enable_gradient_checkpointing=True \
    actor_rollout_ref.actor.fsdp_config.param_offload=False \
    actor_rollout_ref.actor.fsdp_config.optimizer_offload=False \
    actor_rollout_ref.rollout.log_prob_micro_batch_size_per_gpu=32 \
    actor_rollout_ref.rollout.tensor_model_parallel_size=2 \
    actor_rollout_ref.rollout.name=sglang \
    actor_rollout_ref.rollout.mode=async \
    actor_rollout_ref.rollout.gpu_memory_utilization=0.5 \
    actor_rollout_ref.rollout.n=16 \
    actor_rollout_ref.ref.log_prob_micro_batch_size_per_gpu=32 \
    actor_rollout_ref.ref.fsdp_config.param_offload=True \
    actor_rollout_ref.rollout.trace.backend=mlflow \
    actor_rollout_ref.rollout.trace.token2text=True \
    algorithm.use_kl_in_reward=False \
    trainer.critic_warmup=0 \
    trainer.logger='["console","mlflow"]' \
    trainer.project_name='gsm8k_tool-agent' \
    trainer.experiment_name='qwen2.5-3b_function_rm-gsm8k-sgl-tool-agent-verify-n16' \
    trainer.n_gpus_per_node=8 \
    trainer.nnodes=1 \
    trainer.save_freq=-1 \
    trainer.test_freq=20 \
    trainer.total_epochs=15 \
    data.train_files=$HOME/data/gsm8k/train.parquet \
    data.val_files=$HOME/data/gsm8k/test.parquet \
    actor_rollout_ref.rollout.multi_turn.tool_config_path="$PROJECT_DIR/examples/sglang_multiturn/config/tool_config/gsm8k_tool_config.yaml"
```

### 2.5 关键训练参数解读

| 参数 | 值 | 必要性 | 说明 |
|------|----|--------|------|
| `data.return_raw_chat` | `True` | 必需 | 启用聊天格式，Agent Loop 需要原始对话历史 |
| `actor_rollout_ref.rollout.mode` | `async` | 必需 | 异步 rollout 模式，支持多轮交互 |
| `algorithm.adv_estimator` | `grpo` | 推荐 | GRPO 不需要 Critic 网络，简化训练流程 |
| `actor_rollout_ref.rollout.n` | `16` | 推荐 | 每个 prompt 生成 16 个 rollout，GRPO 需要多个样本估计优势 |
| `multi_turn.max_assistant_turns` | `5` | 可调 | 限制最大 assistant 回复轮次，避免过长序列 |
| `rollout.tensor_model_parallel_size` | `2` | 根据模型 | 模型较大时需要 tensor parallel 切分 |
| `rollout.gpu_memory_utilization` | `0.5` | 可调 | SGLang KV Cache 显存占比，多轮对话建议不超过 0.5 |
| `actor.use_kl_loss` | `True` | 推荐 | 使用 KL loss 约束策略更新幅度，替代 KL in reward |
| `actor.kl_loss_coef` | `0.001` | 推荐 | KL loss 系数，值越大约束越强 |

## 3. 使用 MLflow 查看训练追踪

训练期间会自动记录 Agent 的工具调用详情，便于调试和分析模型行为。

### 3.1 启动 MLflow UI

```bash
# 训练完成后（或训练期间）启动 MLflow UI
mlflow ui -h 0.0.0.0 -p 5000 --backend-store-uri sqlite:////tmp/mlruns.db

# 在浏览器中访问
# http://<your_ip>:5000
```

### 3.2 可查看的信息

在 MLflow UI 中可以查看以下内容：

- **完整对话历史** — 每个 rollout 的多轮对话全记录
- **LLM 生成内容** — 模型每一轮的推理过程和输出文本
- **工具调用参数** — 模型传递给工具的具体参数
- **工具返回结果** — 工具执行后返回的奖励值或反馈内容
- **Token 详情** — 每轮生成的 token 数量和内容（需启用 `trace.token2text=True`）

## 4. 训练过程中的正常现象

在训练初期，你可能会观察到以下现象，这些都是正常的：

1. **"Failed to decode tool call" 警告** — 未经训练的模型可能生成格式不正确的工具调用标签（如缺少闭合标签、JSON 格式错误等）。随着训练进行，模型会逐步学会正确的工具调用格式，该警告频率会显著降低。

2. **初始奖励低** — 训练开始时模型可能不会主动调用工具，或者调用工具但传入错误的答案。GRPO 通过对比同一 prompt 的多个 rollout 来估计优势，只要部分 rollout 能获得正确奖励，训练就能正常进行。

3. **KV Cache 利用率波动** — 多轮对话的长度不固定（取决于模型何时停止调用工具），因此 SGLang 的 KV Cache 显存使用会出现波动，这是预期行为。

4. **训练速度较单轮慢** — 多轮 Agent 训练需要在推理过程中执行工具调用和多次 LLM 生成，rollout 阶段耗时会明显长于单轮训练。

## 5. 少量 GPU 的配置调整

如果 GPU 数量少于 8 个，需要相应调整并行参数和 batch size。

### 4 x GPU 配置

```bash
trainer.n_gpus_per_node=4 \
actor_rollout_ref.rollout.tensor_model_parallel_size=1 \
data.train_batch_size=128 \
actor_rollout_ref.rollout.n=8 \
actor_rollout_ref.actor.ppo_mini_batch_size=128
```

### 2 x GPU 配置

```bash
trainer.n_gpus_per_node=2 \
actor_rollout_ref.rollout.tensor_model_parallel_size=1 \
data.train_batch_size=64 \
actor_rollout_ref.rollout.n=4 \
actor_rollout_ref.rollout.gpu_memory_utilization=0.4 \
actor_rollout_ref.actor.ppo_mini_batch_size=64
```

> **注意**：减少 GPU 数量和 batch size 会影响 GRPO 的优势估计质量（更少的 rollout 用于组内对比）。如果训练效果不理想，可以尝试增加 `rollout.n` 或增大 `train_batch_size`。

## 6. 常见问题排查 (FAQ)

### Q: 训练启动后卡住不动

- 检查 GPU 数量是否与 `trainer.n_gpus_per_node` 配置一致
- 确认数据文件路径正确，`train.parquet` 和 `test.parquet` 文件存在
- 检查模型路径是否可以正常下载（`Qwen/Qwen2.5-3B-Instruct`）
- 查看日志中是否有 NCCL 通信错误，可能需要设置 `NCCL_DEBUG=INFO` 排查

### Q: OOM（显存不足）

按优先级依次尝试以下调整：

1. 降低 `gpu_memory_utilization`（如从 0.5 降到 0.4）
2. 减少 `train_batch_size`
3. 减少 `max_response_length`
4. 增加 `tensor_model_parallel_size`（需要更多 GPU）
5. 开启 `actor.fsdp_config.param_offload=True`（会降低训练速度）

### Q: 奖励一直为 0

- 检查工具配置中的 `function.name`（如 `calc_gsm8k_reward`）是否与数据中 `tools_kwargs` 的 key 一致
- 检查数据中 `extra_info.tools_kwargs` 是否包含正确的 `ground_truth`
- 用 MLflow 查看模型是否在生成工具调用 — 如果模型没有调用工具，奖励将始终为 0
- 确认 `extra_info.need_tools_kwargs` 设置为 `True`

### Q: 训练不收敛

- 确保 `data.return_raw_chat=True`（否则 Agent Loop 无法正常工作）
- 确保 `rollout.mode=async`（多轮推理需要异步模式）
- 检查日志中 `tokenization_sanity_check` 是否有 warning（可能存在 tokenization 不一致）
- 减小学习率（如从 `1e-6` 降到 `5e-7`）
- 增加 rollout 数量 `n`（更多样本有助于 GRPO 优势估计）
- 检查 `kl_loss_coef` 是否过大（过强的 KL 约束会阻碍模型学习新行为）

### Q: 工具调用格式错误

- 检查 `multi_turn.format` 是否与模型匹配 — Qwen 系列模型通常使用 `hermes` 格式
- 可以通过 MLflow trace 查看模型具体生成的内容，判断是格式问题还是内容问题
- 如果模型持续无法生成正确格式，考虑先用 SFT 在少量工具调用示例上微调

## 7. 进阶：添加 Interaction 交互

在基础工具 Agent 之上，可以添加 Interaction（交互）组件，为模型提供额外的环境反馈。例如，在 GSM8K 场景中，Interaction 可以在每轮对话后为模型提供提示信息。

### 7.1 交互配置

交互配置文件位于 `examples/sglang_multiturn/config/interaction_config/gsm8k_interaction_config.yaml`：

```yaml
interaction:
  - name: "gsm8k"
    class_name: "verl.interactions.gsm8k_interaction.Gsm8kInteraction"
    config: {}
```

### 7.2 使用带 Interaction 的训练配置

```bash
PROJECT_DIR="$(pwd)"
CONFIG_PATH="$PROJECT_DIR/examples/sglang_multiturn/config"

python3 -m verl.trainer.main_ppo \
    --config-path="$CONFIG_PATH" \
    --config-name='gsm8k_multiturn_grpo_w_interaction' \
    actor_rollout_ref.rollout.multi_turn.interaction_config_path="$PROJECT_DIR/examples/sglang_multiturn/config/interaction_config/gsm8k_interaction_config.yaml" \
    # ... 其他参数同基础配置
```

> **注意**：带 Interaction 的配置文件 `gsm8k_multiturn_grpo_w_interaction.yaml` 使用 `max_user_turns: 5`（而非 `max_assistant_turns`）来限制轮数，因为 Interaction 会在 user 侧注入额外消息。
